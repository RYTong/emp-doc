我们的RYTL脚本库文件名称中会包含版本号，例如：RYTL_1.0.3.lua，模板开发人员在调用版本时需要注意想要调用的脚本库为哪个版本，如果在调用时不填写版本号，那么默认调用当前客户端中已有的最高版本，并且客户端只会保存当前从当前最高版本开始向下的三个版本，例如：当前最高版本为RYTL_1.0.9.lua，那么客户端中会保留RYTL_1.0.9.lua、RYTL_1.0.8.lua、RYTL_1.0.7.lua这三个版本，如果模板开发人员在调用时不写版本号，如RYTL.lua，则默认调用当前的最高版本RYTL_1.0.9.lua。

此文档中只介绍当前最高版本的脚本库接口。

目的

为了便于模板开发人员使用，更加容易简单的编写Lua脚本，我们将封装一套Lua框架。我们将在这个框架中封装不同方法，满足开发需求。

4.1章节与4.2章节定义的Lua库与Lua事件相当于JavaScript语法及API，这些基础的API，在Lua脚本中可以正常调用；而RYTL.lua库则相当于jQuery库，RYTL.lua的出现仅仅是为了让开发人员节省更多的时间，能够更方便的调用，与4.1、4.2章节定义内容就像jQuery与JavaScript的关系一样。

背景

我们封装此套框架以Lua语法为基础，以jQuery库（开源JavaScript函数库）为参考。

实现

模仿jQuery库的功能分类，方法名等。在jQuery使用中，调用jQuery()为构造函数返回jQuery对象，并使用$()代替，我们在Lua中采取类似的使用方法，使用RYTL作为元表（metatable）名，一些基础选取方法都将在这个元表中定义。

注意：在RYTL库中通过方法（例：RYTL:get(…)）返回的对象列表与基础API（例：document:getElementsByName(…)）返回的对象列表不同。我们在RYTL库中为了让每个控件能够支持更多的方法（例如；动画），对很多方法都进行了二次封装。

RYTL库也可完全不使用。

ryt = new RYTL();
-- 此处想从页面中获取出name="password"的控件。下面两句虽然都可以获取到想要的控件列表，但是返回的结果对象并不相同。eles1中的每一个元素为Control类对象（RYTL中定义的类），这些元素可以调用的方法详见Control类中定义；eles2中每一个元素为当前页面中的每一个节点对象（等同于Control.control），这些节点对象是什么类型脚本不需要关心，完全由客户端平台自己定义，这些元素对象可调用的方法为4.1.3章节的elementFunction中的所有Fucntion。
local eles1 = ryt:get{name="password"};
local eles2 = document:getElementsByName("password");

控件选取

允许Lua脚本在当前显示页面中或者符合一定条件的控件队列

Function Syntax Parameter Info
get :get(property) 指定的属性组合（table类型） 返回符合给定属性条件的所有控件（table类型）。注：table中每一个元素为RYTL库中的Control对象。例如：ryt:get{name="info", class="inforcss"}

Function Syntax Parameter Info

网络接口

支持同步传输与异步传输，允许指定回调的Lua函数

Function Syntax Parameter Info
post :post(header, url, data, callback, parameters, synchronous) url：接口地址（不包含服务器域名地址），必填。例：/bank/getui?
data：提交HTTP body内容，可选。（字符串/nil）例：name=lilei&value=ham
callback：回调方法，可选。（函数名/函数体/nil）
parameters : 回调函数传入的参数（responseCode, resposeBody, responseHeader三个key作为保留关键字不可使用）
synchronous：true/false，指定当前操作为同步还是异步，必填。		 callback回调函数的定义必须在post方法调用前，必须且只能声明一个参数，当请求成功该参数表示后台响应报文，不成功则返回错误详情；当此函数为nil时，则该方法返回后台响应报文或请求失败信息。
发起异步请求时必须指定回调函数。
