1 概述

在EWP 5.3.7的版本中，优化了EWP管理平台，将管理平台从EWP中分离出来，并将其分成两部分：EAC（页面控制台）和ADMIN（服务插件）。由于变更内容较多，所以单独写一篇快速使用文档，来介绍如何快速的完成改版前功能的配置。 PS. 如果想使用扩展功能和参数详解，请参考《EWP管理平台使用文档》 和参数详解

2 EAC快速部署

EAC的部署方法与其他APP（ebank、emas）的部署方法相似，操作步骤为：

  1. 获取代码
  2. 编译
  3. 配置
  4. 启动

2.1 获取代码

将代码获取到本地： EAC的代码路径：https://192.168.64.239/hg/proj/emp/ewp_admin 提示：获取代码前，请先申请代码权限

2.2 编译

从服务器上获取EAC的代码后，放到本地

  1. 在ewp_admin路径下，执行./configure如需要调试log，则可加上debug指令./configure --with-debug
  2. make

2.3 配置

EAC有两个配置文件：eac.conf 和 admin_menu.conf。

2.3.1 eac.conf

文件路径： ewp_admin/config/eac.conf 参考样例：

{app_urls, [{"emas", "192.168.64.127:4443"},
            {"ebank", "192.168.64.127:4000"},
            {"unit-test", "ebank", "192.168.64.127:4009"}]
}.

配置方式： EAC访问APP的地址配置，只有正确的配置访问APP

{app_urls, [{AppName, AppUrl}, 
         {AppAlias, AppName, AppUrl}]
}.

AppName：应用名，参数类型：string() AppAlias：应用别名，如"unit-test"，使用别名"unit-test"登录，但是实际使用的应用名为"ebank"，参数类型：string() AppUrl：访问应用的URL，参数类型：string() 特别解释：AppUrl -> "IP:Port" 　　IP：访问应用的服务器IP地址 　　Port：参考访问应用yaws.conf中的port配置

2.3.2 admin_menu.conf

文件路径：ewp_admin/config/admin_menu.conf 文件功能：首页菜单
此配置如无扩展需求，无需更改。如有扩展需求，请参照《EWP管理平台使用文档》

2.4 启动

在ewp_admin路径下，执行脚本./iewp

3 ADMIN插件

3.1 ADMIN插件介绍

ADMIN插件提供的是EWP管理后台逻辑处理功能。它依赖于EWP启动，处理EAC通过HTTP传输的请求，再通过HTTP将处理结果返回给EAC。可支持功能扩展。

3.2 ADMIN插件部署

3.2.1 安装

ADMIN插件已预装在EWP上，安装EWP 5.3.7版本后的安装包即可。

3.2.2 配置

3.2.2.1 ewp.conf

ADMIN插件是在EWP启动中进行加载的，所以，需要在相应的APP的ewp.conf配置文件中追加配置选项。
文件路径：APP/config/ewp.conf （APP：ebank、emas、unit-test……） 参考样例：

{ewp_addon, [
　　{admin,[ 
        %default is ewp_install_dir/addon/admin/config/admin_url_map.conf
        %{map_conf, "/var/www/apps/APP/config/admin_url_map.conf"}
    ]}
]}.

map_conf : ADMIN映射规则配置文件路径，可指定文件路径，如不配置此属性， 则使用默认路径：Ewp_install_dir/addon/admin/config/admin_url_map.conf

3.2.2.2 yaws.conf

每一个APP的ADMIN插件的service是不一样的，所以在yaws.conf中需要添加插件的配置。 文件路径：APP/config/yaws.conf 参考样例：

<server admin>
    port = 4000
    listen = 0.0.0.0
    docroot = public/admin
    appmods = </admin, admin_controller></, error_url>
    auth_skip_docroot = true
    <opaque>
        admin = true
    </opaque>
</server>

server的名字为 admin port :监听的端口（这里的端口同时也用在eac.conf的URL配置中）。 docroot :server使用的根目录，一般用来指定页面所存放的位置。 appmods :yaws回调模块。 auth_skip_docroot :是否跳过寻找docroot文件夹的.yaws_auth文件 opaque/admin = true : 管理平台使用的标识。

3.2.2.3 admin_url_map.conf

ADMIN插件有一个重要的配置文件：admin_url_map.conf。它是ADMIN插件功能的接口映射规则。此配置文件路径是在APP的ewp.conf中配置的，在上面(配置文件:ewp.conf)已经说明。 文件路径： APP/config/admin_url_map.conf
默认配置路径： Ewp_install_dir/addon/admin/config/admin_url_map.conf
如无功能扩展，可以无需配置此文件。

3.2.3 启动

在APP路径下，执行脚本./iewp
