1 Android构建

1.1 Android Build环境配置

  1. 安装JDK
  2. 安装Android SDK
  3. 安装maven
  4. 需要将依赖统一管理，则安装nexus

1.2 Android构建步骤

  1. 需要构建的项目内已放置pom.xml文件(pom.xml文件配置请参考Maven构建工具)
  2. 下载离线资源，并按照客户端要求将离线资源放置在offline目录下(如客户端没有离线资源可跳过当前步骤)
  3. 在配置pom.xml文件根目录下执行命令
    • 子项根目录下执行mvn clean install，将子项目安装的本地的mvn仓库，供父项目使用
    • 父项目根目录下执行mvn clean package，生成apk文件
  4. 上传至企业应用商店(当前通过脚本实现上传，没有企业商店可忽略)

1.3 Android单元测试

  1. 编写android单元测试用例
  2. 将编写好的测试用例放在源码根目录下的test文件夹下
  3. 执行命令mvn test，或者mvn package打包过程中执行测试用例

更详细的单元测试请参考maven使用文档

1.4 Android构建脚本示例

#!/usr/bin/env node
//脚本参数获取
var argv = require('optimist')
    .usage('Usage: $0 --name [emp|emas]')
    .default({
        name: 'emp', host: '192.168.64.127', port: '4000'
    })
    .demand('name')
    .describe('name', 'build golad')
    .describe('zip', 'Zip file generated [y|n]')
    .describe('version', 'upload emas version')
    .describe('host', 'Offline resources host')
    .describe('port', 'Offline resources port')
    .argv;
//导入依赖库
var fs = require('fs');
var unzip = require('unzip');
var process = require('child_process');
var async = require('async');

var uploadSh = 'node ./assets/autobuild/generAppInfo.js --appid android_test_auto_04141542 --label android_test_auto_all --getVersionFrom ./androidVersion --xmlPath ./AndroidManifest.xml --appPath ./' + argv.name + 'ApkFile/appInfo.xml';
if(argv.version !== undefined){
  uploadSh = uploadSh + ' --setVersion ' + argv.version;
}

var MakeShell = function(pathShell, eventFunction) {
    process.exec(pathShell, function(error, stdout, stderr) {
          console.log('stdout: ' + stdout);
          console.log('stderr: ' + stderr);
        if (error) {
            console.log(error);
        } else {
            eventFunction(error);
        }
    });
};

var MovePath = function(goldPath, oldFilePath, newFilePath, eventFunction) {
    fs.exists(goldPath, function(exists) {
        if (exists) {
            fs.rename(oldFilePath, newFilePath, function(err) {
                eventFunction(err, 'the goldpath is found!');
            });
        } else {
            fs.mkdir(goldPath, function(err) {
                fs.rename(oldFilePath, newFilePath, function(err) {
                    eventFunction(err, 'the goldpath is not found!\ncreat the ' + goldPath);
                });
            });
        }
    });
};

//解压文件
var unzipFile = function(fileName, unfilePath, eventFunction){
  var unzipFile = fs.createReadStream(fileName);
  unzipFile.pipe(unzip.Extract({
      path: unfilePath
  }));
  unzipFile.on('end', function(){
    eventFunction();
  });
};

function done(err, results){
  if(err){
    throw err;
  }
  console.log('results: %j', results);
}
//同步执行异步函数
async.series([
  // 离线资源预置
  function(next){
    MakeShell('rm -rf ./assets/offlines/*', next);
  },
  //下载离线资源
  function(next){
    MakeShell('node ./assets/autobuild/downloader.js --platform android --host ' + argv.host + ' --port ' + argv.port, next);
  },
  //解压文件
  function(next){
    unzipFile('file.zip', './assets/offlines', next);
  },
  //执行mvn构建android apk文件
  function(next){
    MakeShell('cd ./EMP/RYTTrack && mvn clean install && cd ../ && mvn clean install && cd ../ViewServer && mvn clean install && cd ../ && mvn clean package', next);
  },
  // 压缩文件，供上传企业应用商店使用
  function(next){
    MovePath('./' + argv.name + 'ApkFile', './target/Emp.apk', './' + argv.name + 'ApkFile/Emp.apk', function() {
        if (argv.zip === 'y') {
            MakeShell(uploadSh, function() {
                MakeShell('zip  android_rebuild.zip ./' + argv.name + 'ApkFile/*', function() {
                    next();
                });
            });
        } else {
            next();
        }
    });
  }
], done);

2 iOS构建

2.1 iOS Build环境配置

  1. xcode5以上版本
  2. mac系统环境

2.2 iOS构建步骤

  1. build FakeFramework(生成框架包头文件)
  2. 执行xcodebuild命令,编译出APP文件
    xcodebuild -project APP.xcodeproj -sdk %s -target %s -configuration Release clean build
  3. 操作project.pbxproj复位许可文件
    def resetXcodeProjConfigAboutProvisioning():
 array = []
 filePath = '%s/APP/APP.xcodeproj/project.pbxproj'%currentPath
 with codecs.open(filePath,'r','utf-8') as the_file:
     for each_line in the_file:
         if each_line.find('PROVISIONING_PROFILE[sdk=iphoneos*]') >= 1:
             each_line = "               \"PROVISIONING_PROFILE[sdk=iphoneos*]\" = \"\";\n"
             array.append(each_line)
         elif each_line.find('PROVISIONING_PROFILE =') >= 1:
             each_line = "               PROVISIONING_PROFILE = \"\";\n"
             array.append(each_line)
         else:
             array.append(each_line)
 codecs.open(filePath,'w','utf-8').writelines(array)
  4. 配置私有的Config.h文件(备注:因为一份代码需要build出4个应用),将配置好的私有Config.h文件写入xcode挂载的Config.h文件
  5. 修改APP-Info.plist文件(CFBundleIdentifier、CFBundleDisplayName、CFBundleVersion、CFBundleShortVersionString、CFBundleURLName、CFBundleURLSchemes)
  6. 执行xode命令，生成ipa可安装程序
    xcodebuild clean -project APP.xcodeproj -sdk %s -target APP -configuration Release CODE_SIGN_IDENTITY=%s build
xcrun -sdk %s PackageApplication -v %s -o %s --sign %s --embed %s
  7. 将输出的ipa文件、plist文件和appInfo.xml文件压缩为zip包(如没有企业应用商店上传需求则可以忽略)
