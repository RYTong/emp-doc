1 需求概述

可参见离线协议2.1(H5资源下载) # 需求概述

2 设计

新增离线协议版本 2.1 来支持H5资源更新。

2.1 更新

  1. resource_hash 、 resource_update接口的version参数 及 信道建立时res_ver参数修改为离线版本 2.1。

  2. resource_update接口上传已下载的必选资源和H5资源描述, 上传描述信息格式可参见离线协议2.1(H5资源下载) # 客户端上传描述格式

  3. 服务器返回描述信息格式可参见离线协议2.1(H5资源下载) # 服务器返回描述格式,客户端解析 h5 及 h5_d字段信息

2.2 下载

  1. 根据服务器返回的描述信息中的h5字段信息下载资源，下载时path参数拼接方式同离线2.0版本

  2. 已下载的H5资源保存到h5.desc文件中,保存格式:

    {
 "path/file1" : 
 {
   "r" : "c4f3cffb7d68e63531c60e74611a2d2f569173e7"
   "e" : 1 //如果不需要加密存储,则没有此字段
   "n" : "资源说明"
 }
 }
  1. 客户端增加h5-resources目录保存已下载的H5资源

2.3 删除

根据服务器返回的资源描述中 h5_d 字段信息,从h5-resources目录下删除对应资源并更新h5.desc中的描述信息。

2.4 访问

H5资源的校验及读取可参见lua接口改造

2.5 预置资源

H5预置资源处理与原流程一致. 描述格式可参见离线协议2.1(H5资源下载) # H5资源预置

预置H5资源根目录：offline/h5/

2.6 lua接口改造

2.6.1 offline:checkOfflineFileWithLocalH5(filename)

  • Description:
    检测本地H5离线资源文件是否有效。
    • 有效定义： 客户端本地存在对应文件，并且和本地存储的已下载必选文件描述一致。
  • Parameters:
    1. filename(required)
      • H5资源的文件路径 + 文件名。
  • Return:
    • 类型：Boolean
    • 说明：true表示当前文件有效，false表示当前文件无效

2.6.2 offline:checkOfflineFileWithServerH5(filename)

  • Description:
    检测本地H5离线资源是否有效。
    • 有效定义：客户端本地存在对应文件，并且和本地存储的最新服务器资源描述一致。
  • Parameters:
    1. filename(required)
      • 类型：String
      • 说明：H5资源的文件路径 + 文件名。
  • Return:
    • 类型：Boolean
    • 说明：true表示当前文件有效，false表示当前文件无效

2.6.3 file:readH5(name, type)

  • Description:
    根据给定的文件路径读取离线资源H5文件内容。
  • Parameters:
    1. name(required)
      • 类型: String.
      • 说明: H5资源的文件路径 + 文件名（带后缀）
    2. type(required)
      • 类型: String.
      • 说明: 文件类型（"text"-文本，"image"-图片，"stream"-流媒体）
  • Return:
    • 类型: String/Userdata.
    • 说明: 文件类型为"text"则返回字符串（String），为"image"则返回二进制数据（Userdata类型），文件不存在则返回nil。
