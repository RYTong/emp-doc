国际化设计文档

  • 加密信道错误提示

通常客户端的国际化实现各个平台都会相应的方案.各个平台按照SDK中提供的方案即可.

针对部分内容,需要各平台统一设计方案已达到功能效果的要求.

1 加密信道错误提示

加密信道中的各种错误提示需要通过国际化方案达到项目中可配置的目的. 同时为了防止项目中的配置后,在报错时无法准确定位问题,需要增加错误码. 方案如下:

  1. 增加加密信道报错错误码.如下错误码是根据代码中的错误检测的位置增加的.

    • verifyServerCertificate方法 建立安全通道失败，原因为证书校验失败。 errorCode:9010 9011
    • verifyHmacSha1: 方法 建立安全通道失败，原因为hmac校验失败。 errorCode:9020 9021. 这里有两个错误码,来自于代码中的两处错误判断
    • verifyFinishData方法 建立安全通道失败，原因为finish校验失败。 errorCode:9030 9031
    • TLSHelloAndKeyExchangeReponse 请求回调方法建立安全通道失败，错误代码: %d（本地化时采用建立安全通道失败。此句）. errorCode:由服务器返回

  2. 对报错的错误提示内容,增加国际化配置.如建立安全通道失败，原因为证书校验失败。提示语. 后面增加errorCode:错误码.通过国际化文件中对是否显示错误码的配置控制是否提示错误码.如果是否显示错误码配置为是.则提示错误码，反之不显示.实例代码如下:

     NSString *message = NSLocalizedString(@"建立安全通道失败，原因为证书校验失败。",nil);
 NSString *message1 =  NSLocalizedString(@"是否显示错误码", nil);
 if ([message1 isEqualToString:@"是"]) {
     NSString *errorCode =  @"errorCode:9010";
     message = [NSString stringWithFormat:@"%@%@",message,errorCode];
 }

  3. 其他 关于其他信息的配置,使用标准的国际化配置方案即可.
