Screen adaptation

  • EMP适配设计
    • 横向等比缩放
      • 关于适配算法的选择
    • 字号的缩放
    • 提供灵活的缩放算法

随着手机的机型越来越多,各种机型的屏幕尺寸及分辨率有很大差异.为了降低开发成本,通常需要指定某个屏幕尺寸作为标准屏,其他的屏幕按照这个标准屏做横向比例缩放. 屏幕适配算法是为了能在不同的手机屏幕下显示相同的页面内容而设计的算法.

1 EMP适配设计

1.1 横向等比缩放

EMP 的适配方案采用横向等比缩放的方式,即通过屏幕的宽度的比例对界面元素的大小及坐标进行缩放计算.
如果设置的标准屏为320 480,标准屏上的某一个控件的大小为10050.将其适配到 iphone6(iphone6 的屏幕分辨率为375x667)上的大小为:
宽度: 375 100 / 320;
高度: 375 50 / 320;
比例系数为:375 / 320;

1.1.1 关于适配算法的选择

通常情况下比例系数为小数,因此最后计算的结果也可能包含小数.
通过测试发现带有小数的坐标和 size 会导致界面显示有问题.因此控件的 frame 值需要保证是整形数据.
针对小数取整的方法有两种:四舍五入和向下取整.
四舍五入算法 四舍五入算法可以实现坐标取整的目的,但是可能会导致最后计算的结果大于实际值. 由于客户端的自动布局算法会根据屏幕宽度计算换行,当计算结果大于实际值时,会出现换行的情况. 这将导致在两种屏幕下布局效果的差异. 向下取整 详细取整同样可以解决坐标取整的目的,其计算的结果会比实际值小.从而避免了四舍五入算法中导致的布局效果不统一的问题. 向下取整算法的问题在于其将控件的大小算小后,右侧会出现缝隙,因为其不能完全填满屏幕.

综合上面的问题,我们选择详细取整算法作为我们的适配算法规则.

1.2 字号的缩放

文字字号的缩放才有与坐标的适配大体相同,但是字号的适配采用四舍五入的方式达到取整的问题.

1.3 提供灵活的缩放算法

由于不算的有新的机型的出现,通常我们需要保证主流机型的最佳显示效果.因此适配算法需要能够灵活的配置适配的方案.

如果以 iphone6的屏幕作为标准屏,而其他的机型以此为基准做适配.此时需要对界面元素的坐标及大小的设置以 iphone6的375 667 为基准. 将其适配到 iphone4(320480)的屏幕上. 一个标准屏下100 50的控件. 在 iphone4上的大小为:
宽度: 320 100 / 375;
高度: 320 * 50 / 370;
比例系数: 320 / 375;

因此我们要求在设计时,这个比例系数是可以设置的. 客户端需要提供相应的 API 接口来设置比例系数.

1.4 关于适配算法的封装

考虑到目前的适配算法并不完善,今后有可能会有不同程度的修改,因此建议将此部分算法封装. 例如:

  • (CGFloat)adaptedNumber:(CGFloat)original;// 传入适配前的值,得到适配后的值
    • (CGFloat)restoreAdaptedNumber:(CGFloat)adapted;// 传入适配后的值,得到适配前的值
